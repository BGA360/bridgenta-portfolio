## 2. Verification Guidelines

1. **Grounded Source:** Every number or speedup claim must refer to a specific test run or version-controlled log.
2. **Context Bounding:** Text must explicitly state the testing environment (e.g. `im Pilotbetrieb`, `im Testlauf`) so readers do not assume general product-wide guarantees.
3. **No Hype:** Remove qualitative superlatives ("höchste Performance", "schnellste Integration"). Present raw quantitative values.
