# BECC v2.0 — Changelog

This changelog records the completed changes, additions, validations, and deferrals introduced during the BECC v2.0 engineering lifecycle.

---

## [2.0.0] - 2026-07-14

### Added
*   **Constitutional Knowledge Resolution Architecture**: Design spec for decoupled evaluation loops.
*   **Canonical Data Model**: Schema specs for all operational workspace documents.
*   **Engineering Domain Specifications**: EDS documents for Runtime Orchestrator, Knowledge Resolver, Project Connector, and Validation Engine.
*   **Portfolio Improvement Candidate Register**: Living evidence backlog (`BECC-v2-PORTFOLIO-IMPROVEMENT-CANDIDATE-REGISTER.md`).
*   **Portfolio Improvement Candidate Review Board (PICRB)**: Governance protocol and review reports (`PICRB-001-PORTFOLIO-IMPROVEMENT-REVIEW.md`).
*   **Template Generation Model**: Classification method for Generation 1 (Legacy) and Generation 2 (Evolved) documents.
*   **Assessment Workspaces**: Standardized folders for `AC-001`, `AC-002`, `AC-003`, and `AC-004`.

### Changed
*   **Stewardship Ledger**: Migrated `BECC-ASSESSMENT-LEDGER.md` to support multi-project state tracking and commit verification.
*   **Remediation Positioning**: Switched from raw line-number specs to semantic section anchors.
*   **Relative Pathing**: Standardized on absolute URL link patterns for public-facing documentation.

### Validated
*   **AC-001 (AEOcortex)**: Full audit, remediation, and closure.
*   **AC-002 (Lumina Praxis)**: Repetitive EDR execution.
*   **AC-003 (StarCleaners)**: Workflow repetition and overhead optimization.
*   **AC-004 (BridGenta)**: Direct compliance validation.

### Deferred
*   **Portfolio Evolution Framework**: Automation tools for programmatic template migrations (Deferred to v2.1).
*   **Communication Maturity Classification**: Document grading scales (Deferred to v2.2).
*   **Automatic Template Generation Detection**: AST-based generation identification (Deferred to v2.1).
*   **Adaptive Assessment Profiles**: Decooupled compliance matrices (Deferred to v2.2).
